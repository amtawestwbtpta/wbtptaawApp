Testing Instructions

How to Test the West Bengal OBC Categories Application

Follow these steps to ensure the application is working as expected:

  1. Initial Load Test:
    • Expected Behavior: When you first load the page, you should see a title "West Bengal OBC Categories (66 Valid Before 2010)" and a dropdown labeled "Select an OBC Category:".
    • Expected Behavior: Below the dropdown, the "Selected Category:" display and the "Category Type:" and "Notification No.:" information note should not be visible. They should be hidden.
  2. Dropdown Population Test:
    • Expected Behavior: Click on the dropdown. It should populate with a list of 66 OBC categories, each prefixed with a number (e.g., "1. Baishya Kapali", "2. Bansi Barman", etc.).
    • Expected Behavior: The categories should be sorted alphabetically by their names.
  3. Category Selection Test (Positive Case):
    • Action: Select any category from the dropdown, for example, "Kapali".
    • Expected Behavior:
      • The "Selected Category:" display should immediately become visible and show "Selected Category: Kapali".
      • The information note below it should also become visible.
      • The "Category Type:" in the note should display "OBC-B".
      • The "Notification No.:" in the note should display the full notification number associated with "Kapali".
  4. Another Category Selection Test (Positive Case):
    • Action: Select a different category, for example, "Jolah (Ansari-Momin)*".
    • Expected Behavior:
      • The "Selected Category:" display should update to "Selected Category: Jolah (Ansari-Momin)*".
      • The "Category Type:" in the note should update to "OBC-A".
      • The "Notification No.:" in the note should update to the full notification number for "Jolah (Ansari-Momin)*".
  5. Re-selecting "-- Select --" Test (Negative Case):
    • Action: After selecting a category, re-select the initial "-- Select --" option from the dropdown.
    • Expected Behavior:
      • The "Selected Category:" display should disappear (become hidden).
      • The information note ("Category Type:" and "Notification No.:") should also disappear (become hidden).
  6. Responsiveness Test:
    • Action: Resize your browser window to simulate different screen sizes (e.g., narrow it to mimic a mobile phone, then widen it for a desktop).
    • Expected Behavior: The layout should adjust gracefully. The main card should center on the screen, and the dropdown and info note should scale to fit the available width without horizontal scrolling.
